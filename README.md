# Enchantify

*A living book that turns attention into artifacts.*

Enchantify is a local-first narrative operating system disguised as a magical school story. It runs inside OpenClaw, reads a folder of lore, scripts, logs, and player state, and turns ordinary life into an ongoing urban-fantasy game: scenes, letters, dreams, newspaper issues, Compass Runs, Enchantments, support-character check-ins, and a world that keeps moving between sessions.

The central object is the Labyrinth of Stories: a sentient book that opens to the page your life and the Academy need today.

## What It Is

Enchantify is not a conventional app with one loop. It is a living-book framework:

- **Every interaction opens a Page.** Slice of Life, Conflict, Enchantment, Wonder Compass, Letter, Anchor, Rest, Archive, Bleed, Vellum, Inkrest, Ledger, and Book Jump pages give each moment a purpose.
- **Every Page asks for an answer.** The player may speak, choose, walk, notice, photograph, rest, write, or act in the real world.
- **Every answer leaves proof.** Diary entries, scene ledgers, printed cards, letters, images, audio, Belief changes, story-thread updates, and field notes become the book’s memory.

The result is a game that tries to make daily life feel authored without making it feel fake.

## Core Systems

- **Active play** through OpenClaw and Telegram, using script-gated scene construction and buffered delivery.
- **Story-Field Journal** in `hooks/mission-control.html`, generated by `scripts/mission-control.py`, for live visibility into threads, arcs, pages, support faculty, ledger, health, Compass Runs, Book Jumps, entities, and crons.
- **Belief economy** where NPCs, talismans, threads, objects, and factions gain or spend narrative weight.
- **Living-world simulation** where registry entities take bespoke offscreen actions, spend Belief, and feed playable consequences back into future scenes.
- **Rule of Three choices**: one slice-of-life option, one arc/thread option, and one surprise option.
- **Wonder Compass Runs**: deterministic real-world attention rituals based on Notice, Embark, Sense, Write, and Rest.
- **Enchantments**: photo or description based spells that bridge the Academy into the player’s actual world.
- **Book Jumping**: structured travel into books, with stabilization, return, souvenirs, and Nothing pressure.
- **The Bleed**: an Academy newspaper that digests simulation, health, food, finances, support faculty, external signals, and rumor into print and Telegram.
- **NPC research and outreach**: characters can research Unwritten interests and send letters or messages in their own voices.
- **Support faculty**: Dr. Vellum for longevity/body data, Dr. Inkrest for narrative therapy and mood memory, and Gimble for Actual Budget/SimpleFIN ledger support.
- **Pact and app systems** where factions and talismans can influence configured apps under guardrails.
- **Local art and audio pipeline** for manuscript-style generated images, Draw Things keepalive art, wallpapers, Kokoro/TTS voices, and optional MusicGen.

## Project Shape

```text
AGENTS.md                 Runtime operating rules for the Labyrinth
SOUL.md                   Voice and persona
mechanics/                Game rules, page contracts, dice, routing
lore/                     World, characters, enchantments, threads, registers
scripts/                  Deterministic mechanics and automation
hooks/                    Story-Field Journal, guides, generated surfaces
config/                   Example config plus non-secret tracked defaults
ios/InsideCover/          iOS widget prototype
templates/                Player and setup templates
```

Generated personal state is intentionally ignored where possible: `players/`, live heartbeats, most logs, health/fuel data, Actual Budget credentials, images, and runtime caches.

## Running the Game

The main player-facing instruction is still simple:

```text
open the book
```

But the agent must use the scripts in `AGENTS.md`. Active play is not plain chat. It routes through state readers, scene contracts, choice validation, mechanics preflight, and local delivery scripts.

Useful local commands:

```bash
python3 scripts/mission-control.py
python3 scripts/mission-control.py --serve
python3 scripts/pulse.py
python3 scripts/world-pulse.py
python3 scripts/send_academy_dispatch.py --dry-run
python3 scripts/check-runtime.py
```

## Current Design Direction

Enchantify’s north star is:

> Every interaction opens a page. Every page has a purpose. Every purpose leaves proof.

The interface should feel like a magical field journal or annotated storybook: sparse pen-and-ink linework, watercolor washes, aged parchment, lush marginalia, stamps, seals, handwritten notes, and selective jewel-like pops of color.

Future mobile work lives under the idea of **Enchantify Glass**: a pocket Story-Field Journal for iPhone and iPad, likely with Today, Compass, Library, and Desk tabs.

## Privacy and Safety

Enchantify is deliberately local-first. It can touch intimate data: health logs, food logs, mood check-ins, finances, location anchors, calendar events, and personal story memory. Keep real credentials and personal state out of Git. Use example config files and local secret stores.

Support characters may give useful everyday guidance, but they should not replace doctors, therapists, lawyers, financial advisors, or emergency care.

## Status

This is an active personal project under heavy development. The codebase contains both stable systems and living experiments. The safest way to work on it is:

1. Read the relevant script and mechanic file first.
2. Keep changes deterministic where possible.
3. Avoid generic boilerplate in player-facing narrative.
4. Preserve the living-book metaphor.
5. Let the book keep proof.
